# Industry Readiness & Open Source Internship Program

## Overview
This initiative aims to bridge the gap between academic learning and real-world software engineering practices.

Students undergo structured training and subsequently contribute to active open-source and community-driven projects under mentorship.

The program focuses on building practical skills in software development, project collaboration, engineering processes, cybersecurity, research support, and modern cloud-native technologies.

---

## Program Structure

### Phase 1: Foundation Training (First 2 Months)
Students receive hands-on training in:
- Programming languages relevant to project assignments
- Git and GitHub workflows
- Branching strategies
- Pull Requests and Code Reviews
- Software Development Life Cycle (SDLC)
- Agile Methodology
- JIRA Project Management
- Requirement Engineering
- Documentation Practices
- Cybersecurity Fundamentals
- Open Source Contribution Process
- Team Collaboration Practices

### Phase 2: Project Contribution (Month 3 Onwards)
After successful completion of training, students begin contributing part-time to active projects under mentor supervision.

Students may work on:
- Research support activities
- AI Agent development
- Function-as-a-Service (FaaS) platforms
- Community Infrastructure projects
- SDLC automation tools
- Open-source software initiatives
- Remote services and cloud-native applications

See **PROJECTS.md** for the current list of active projects.

---

## Learning Outcomes
By the end of the program, participants will be able to:
- Work effectively in software development teams
- Use Git and GitHub professionally
- Participate in code reviews
- Understand and follow SDLC processes
- Manage tasks using Agile methodologies
- Analyze and implement requirements
- Write technical documentation
- Apply cybersecurity best practices
- Contribute to open-source projects
- Design, develop, test, and deploy software solutions

## Benefits for Students

### Real-World Project Exposure
Gain hands-on experience by contributing to active projects rather than working only on academic exercises.

### Industry Engineering Practices
Learn and practice:
- GitHub Workflows
- Pull Requests
- Code Reviews
- Agile Development
- Sprint Planning
- Documentation Standards
- Software Quality Practices

### Open Source Contributions
Build a public portfolio through meaningful contributions to open-source projects.

### Team Collaboration
Work with mentors, researchers, developers, and fellow contributors in a collaborative environment.

### Project Ownership
Experience the complete development lifecycle from requirement analysis to deployment.

### Academic Project Support
Where applicable, students may extend their contributions into final-year academic projects.

### Career Preparation
Develop practical experience that strengthens resumes and improves interview readiness.

### Experience Certificate
Participants who successfully contribute to project activities may receive an experience certificate recognizing their contributions.

## Program Progression

**Level 1 – Trainee Engineer**
- Complete training modules
- Perform guided exercises
- Learn development workflows

**Level 2 – Associate Contributor**
- Contribute to project tasks
- Participate in code reviews
- Collaborate with project teams

**Level 3 – Project Contributor**
- Own features and deliverables
- Assist junior contributors
- Participate in design discussions

**Level 4 – Technical Lead Intern**
- Lead small teams
- Review contributions
- Coordinate project activities
- Support mentoring efforts
